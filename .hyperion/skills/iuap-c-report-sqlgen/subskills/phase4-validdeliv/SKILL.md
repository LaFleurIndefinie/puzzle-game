# 阶段四：校验与交付

> 本技能是报表 SQL 生成流程的最后阶段，负责校验 SQL 并完成交付。

## 阶段目标

- AI 自检（按清单逐项检查）
- 执行 db_query.py 校验
- 交付最终产物

## 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| SQL 文件路径 | 阶段三传递 | 未校验的完整 SQL |
| 字段映射表 | 阶段二传递 | 完整的字段类型分析 |
| 说明文档骨架 | 阶段三传递 | 说明文档初稿 |

## 步骤 7：AI 自检（强制执行）

### 自检清单

AI 必须**逐项打勾检查**，不允许跳过：

| 序号 | 检查项 | 状态 |
|:--:|--------|:--:|
| 1 | ✅ 所有表名都使用 `schema.tableName` 格式？ | ▢ |
| 2 | ✅ 特征表使用字段自身的 `tableName`（不是特征组虚拟表名）？ | ▢ |
| 3 | ✅ WHERE 子句**必须**包含租户过滤：`ytenant_id = 'var$(租户id)'`？ | ▢ |
| 4 | ✅ 除了租户过滤，只添加了用户明确要求的筛选条件？ | ▢ |
| 5 | ✅ 所有枚举字段都已添加 CASE WHEN 转换？ | ▢ |
| 6 | ✅ 所有字段都来自元数据，没有 AI 臆造？ | ▢ |
| 7 | ✅ 字段别名使用双引号包裹中文？ | ▢ |
| 8 | ✅ JOIN 顺序符合规范（主表 → 特征 → 平行 → 参照 → 子表）？ | ▢ |
| 9 | ✅ SQL 语法正确，引号、逗号、括号都已闭合？ | ▢ |
| 10 | ✅ 如果是语义脚本参数，使用 Freemarker `<#if>` 语法？ | ▢ |

### 自检不通过处理

如果有任何检查项不通过：
1. 修正 SQL 中的问题
2. 重新执行自检
3. 直到全部通过才能进入步骤 8

## 步骤 8：执行校验（硬性门禁）

### 执行校验命令

```bash
# 执行 SQL 校验
python scripts/db_query.py --sql-file report_sql_output/{报表名}.sql
```

### 校验说明

- `var$(租户id)` 由脚本自动替换为 `YONBIP_TENANT_ID` 进行校验
- SQL 源文件不变
- `database.enabled: false` 时不得声称已校验

### 结果判定

| 结果 | 处理 |
|------|------|
| **校验通过** | 显示查询结果行数 → 进入步骤 9 |
| **校验失败** | 显示错误信息 → 修正 SQL → 回到步骤 7 重新自检 |

### 常见错误修正

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| 表不存在 | 表名/schema 错误 | 检查 `schema.tableName` 格式 |
| 字段不存在 | dbColumnName 错误 | 从元数据重新获取正确字段名 |
| 语法错误 | SQL 语法问题 | 修复 SQL 语法 |

## 步骤 9：交付输出

### 交付物规范

| 文件 | 内容 | 要求 |
|------|------|------|
| `{报表名}_{时间戳}.sql` | 仅 SQL | 无 Markdown 围栏，可直接复制执行 |
| `{报表名}_说明_{时间戳}.md` | 说明文档 | 按模板填写，不内嵌完整 SQL |
| `{报表名}_特征表校验_{时间戳}.sql` | 诊断 SQL | 仅当涉及 character_define 时生成 |
| `报表需求确认单_{报表名}_{时间戳}.xlsx` | 需求确认单 | 按模板格式生成，包含业务对象、字段、JOIN、条件四部分 |

> **时间戳格式**：`YYYYMMDD_HHmmss`（例如：`20260428_143052`）

### 生成确认单 Excel

校验通过后，使用 `gen_confirm_excel.py` 生成需求确认单：

```bash
python .claude/skills/iuap-c-report-sqlgen/subskills/phase4-validdeliv/scripts/gen_confirm_excel.py \
    --sql-file report_sql_output/{报表名}.sql \
    --entities output/entities.json \
    --output report_sql_output/报表需求确认单_{报表名}_{时间戳}.xlsx \
    --report "{报表名}"
```

**确认单内容**：
- 一、业务对象确认：所有涉及的表（schema、tableName、URI、说明）
- 二、输出字段确认：13个字段（数据库字段路径、来源表、补充说明）
- 三、JOIN关系说明：主查询和子查询中的 JOIN 关系
- 四、查询条件说明：WHERE 条件 + GROUP BY 字段

### 输出目录结构

```
report_sql_output/
├── {报表名}_20260428_143052.sql
├── {报表名}_说明_20260428_143052.md
├── {报表名}_特征表校验_20260428_143052.sql  # 可选
└── 报表需求确认单_{报表名}_20260428_143052.xlsx
```
